## POST /api/create-project

Create a new project and associate contributors.

### Request Body

<ParamField body="title" type="string" required>
  Project title
</ParamField>

<ParamField body="description" type="string">
  Project description
</ParamField>

<ParamField body="owner" type="string" required>
  Email address of the project owner
</ParamField>

<ParamField body="maintainers" type="array" required>
  Array of email addresses of project maintainers
</ParamField>

<ParamField body="contributors" type="array" required>
  Array of email addresses of project contributors
</ParamField>

<ParamField body="tasks" type="array">
  Array of task IDs (optional, usually empty on creation)
</ParamField>

<ParamField body="contributions" type="array">
  Array of contribution records (optional)
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

<ResponseField name="project" type="object">
  Created project object
  
  <Expandable title="project properties">
    <ResponseField name="_id" type="string">
      Project's unique identifier
    </ResponseField>
    
    <ResponseField name="title" type="string">
      Project title
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Project description
    </ResponseField>
    
    <ResponseField name="owner" type="string">
      Email address of the project owner
    </ResponseField>
    
    <ResponseField name="maintainers" type="array">
      Array of maintainer email addresses
    </ResponseField>
    
    <ResponseField name="contributors" type="array">
      Array of contributor email addresses
    </ResponseField>
    
    <ResponseField name="tasks" type="array">
      Array of task IDs
    </ResponseField>
    
    <ResponseField name="contributions" type="array">
      Array of contribution records
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO timestamp of project creation
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO timestamp of last update
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl -X POST https://your-domain.com/api/create-project \
  -H "Content-Type: application/json" \
  -d '{
    "title": "TeamUp Mobile App",
    "description": "A mobile application for team collaboration",
    "owner": "owner@example.com",
    "maintainers": ["owner@example.com", "maintainer@example.com"],
    "contributors": ["owner@example.com", "maintainer@example.com", "dev1@example.com"],
    "tasks": [],
    "contributions": []
  }'
```

### Example Response

```json
{
  "message": "Project created successfully!",
  "project": {
    "_id": "507f191e810c19729de860ea",
    "title": "TeamUp Mobile App",
    "description": "A mobile application for team collaboration",
    "owner": "owner@example.com",
    "maintainers": ["owner@example.com", "maintainer@example.com"],
    "contributors": ["owner@example.com", "maintainer@example.com", "dev1@example.com"],
    "tasks": [],
    "contributions": [],
    "createdAt": "2024-01-15T10:30:00.000Z",
    "updatedAt": "2024-01-15T10:30:00.000Z"
  }
}
```

### Error Responses

<ResponseField name="500 Internal Server Error">
  ```json
  {
    "message": "Error creating project"
  }
  ```
</ResponseField>

### Notes

- Automatically updates each contributor's `projects` array with the new project ID
- Extended timeout to 60 seconds for database operations

## POST /api/project

Delete or leave projects.

### Request Body

<ParamField body="userEmail" type="string" required>
  Email address of the user performing the action
</ParamField>

<ParamField body="toDelete" type="array">
  Array of project IDs to delete (owner only)
</ParamField>

<ParamField body="toLeave" type="array">
  Array of project IDs to leave (contributors)
</ParamField>

### Response

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

### Example Request (Delete)

```bash
curl -X POST https://your-domain.com/api/project \
  -H "Content-Type: application/json" \
  -d '{
    "userEmail": "owner@example.com",
    "toDelete": ["507f191e810c19729de860ea"]
  }'
```

### Example Request (Leave)

```bash
curl -X POST https://your-domain.com/api/project \
  -H "Content-Type: application/json" \
  -d '{
    "userEmail": "contributor@example.com",
    "toLeave": ["507f191e810c19729de860ea"]
  }'
```

### Example Response

```json
{
  "message": "success"
}
```

### Error Responses

<ResponseField name="400 Bad Request">
  ```json
  {
    "message": "No action performed"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  ```json
  {
    "message": "Error in deletion process"
  }
  ```
  or
  ```json
  {
    "message": "Error in leaving process"
  }
  ```
</ResponseField>

### Notes

**Delete Operation (`toDelete`):**
- Removes all tasks associated with the project
- Removes task references from all users
- Removes project from all contributors
- Deletes the project permanently

**Leave Operation (`toLeave`):**
- Removes user from project contributors and maintainers
- Deletes tasks where user is the only assignee
- Removes user from tasks with multiple assignees
- If last maintainer leaves, owner is automatically added as maintainer

</ParamField>

  Whether the request was successful


### Example Response

```json
{
```

```json
  "data": {
    "title": "TeamUp Mobile App",
    "totalTasks": 24,

    "activeContributors": 5
```

  ```json
    "success": false,
    "data": null,
    "message": "Project not found",
  ```
