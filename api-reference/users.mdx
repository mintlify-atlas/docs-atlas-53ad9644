## GET /api/user-search

Search for users by email address using regex pattern matching.

### Query Parameters

<ParamField query="t" type="string" required>
  Search term for email address (case-insensitive regex matching)
  
  Example: `"john"` will match `"john@example.com"`, `"johnny@test.com"`, etc.
</ParamField>

### Response

Returns an array of user objects matching the search criteria, sorted by email address, limited to 10 results.

<ResponseField name="users" type="array">
  Array of user objects
  
  <Expandable title="user properties">
    <ResponseField name="_id" type="string">
      User's unique identifier
    </ResponseField>
    
    <ResponseField name="name" type="string">
      User's full name
    </ResponseField>
    
    <ResponseField name="emailId" type="string">
      User's email address
    </ResponseField>
    
    <ResponseField name="profilePic" type="string">
      URL to user's profile picture
    </ResponseField>
    
    <ResponseField name="projects" type="array">
      Array of project IDs the user belongs to
    </ResponseField>
    
    <ResponseField name="tasks" type="array">
      Array of task IDs assigned to the user
    </ResponseField>
    
    <ResponseField name="contributions" type="array">
      Array of contribution records
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO timestamp of user creation
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO timestamp of last update
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl https://your-domain.com/api/user-search?t=john
```

  {

    "name": "Johnny Smith",
    "emailId": "johnny.smith@company.com",
    "profilePic": "https://lh3.googleusercontent.com/a/...",

### Error Responses

<ResponseField name="404 Not Found">
  ```json
  {
    "message": "No users found"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  ```json
  {
    "message": "Error fetching users"
  }
  ```
</ResponseField>

### Notes

- Search is case-insensitive
- Uses MongoDB regex matching with `$options: 'i'`
- Results are sorted alphabetically by email address
- Maximum 10 results returned
- Extended timeout to 60 seconds for database operations
- Useful for autocomplete features when adding collaborators to projects

### Search Examples

| Search Term | Matches |
|-------------|----------|
| `john` | `john@example.com`, `johnny@test.com`, `mike.johnson@company.com` |
| `@gmail` | `user@gmail.com`, `test@gmail.com` |
| `dev` | `dev@example.com`, `developer@test.com`, `mike.dev@company.com` |
| `example.com` | `john@example.com`, `jane@example.com` |

## GET /api/userinfo

Get information about the currently authenticated user. This endpoint is also documented in the [Authentication](/api-reference/authentication) section.

### Authentication

Requires valid NextAuth session (Google OAuth).

### Response

Returns the current user's information or creates a new user if authenticated via Google but not yet in the database.

<ResponseField name="user" type="object" or name="array" type="array">
  User object or array containing user object
  
  <Expandable title="user properties">
    <ResponseField name="_id" type="string">
      User's unique identifier
    </ResponseField>
    
    <ResponseField name="name" type="string">
      User's full name from Google profile
    </ResponseField>
    
    <ResponseField name="emailId" type="string">
      User's email address
    </ResponseField>
    
    <ResponseField name="profilePic" type="string">
      URL to user's profile picture from Google
    </ResponseField>
    
    <ResponseField name="projects" type="array">
      Array of project IDs the user belongs to
    </ResponseField>
    
    <ResponseField name="tasks" type="array">
      Array of task IDs assigned to the user
    </ResponseField>
    
    <ResponseField name="contributions" type="array">
      Array of contribution records
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO timestamp of user creation
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO timestamp of last update
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl https://your-domain.com/api/userinfo \
  -H "Cookie: next-auth.session-token=..."
```

### Example Response (Existing User)

```json
[
  {
    "_id": "507f1f77bcf86cd799439011",
    "name": "John Doe",
    "emailId": "john@example.com",
    "profilePic": "https://lh3.googleusercontent.com/a/...",

    "tasks": ["507f191e810c19729de860eb"],
    "contributions": [],
    "createdAt": "2024-01-15T10:30:00.000Z",
    "updatedAt": "2024-01-15T10:30:00.000Z"
  }
]
```

### Example Response (New User Auto-Created)

```json
{
  "_id": "507f1f77bcf86cd799439011",
  "name": "John Doe",
  "emailId": "john@example.com",
  "profilePic": "https://lh3.googleusercontent.com/a/...",
  "projects": [],
  "tasks": [],
  "contributions": [],
  "createdAt": "2024-01-15T10:30:00.000Z",
  "updatedAt": "2024-01-15T10:30:00.000Z"

```

### Error Responses

<ResponseField name="500 Internal Server Error">
  ```json
  {
    "message": "Error fetching User's Info"
  }
  ```
</ResponseField>

### Notes

- Automatically creates a new user record if Google-authenticated user doesn't exist
- Returns existing user data if user is already registered
- Extended timeout to 60 seconds for database operations
- User profile picture is automatically fetched from Google account

## User Model Schema

The User model in TeamUp has the following schema:

```typescript
{
  name: string,              // Required: User's full name
  emailId: string,           // Required, Unique: User's email
  profilePic?: string,       // Optional: Profile picture URL
  projects: string[],        // Array of project ObjectIds
  tasks: string[],          // Array of task ObjectIds
  contributions: any[],     // Array of contribution records
  createdAt: Date,          // Auto-generated timestamp
  updatedAt: Date           // Auto-updated timestamp
}
```

### Indexes

The User collection has the following indexes for optimized queries:

- `emailId` (unique)
- `tasks`
- `projects`

### Common Use Cases

**Find User by Email**
```bash
curl https://your-domain.com/api/user-search?t=user@example.com
```

**Get Current User Info**
```bash
curl https://your-domain.com/api/userinfo \
  -H "Cookie: next-auth.session-token=your-session-token"
```

**Search for Collaborators**
```bash
# Search for users with "@company.com" email
curl https://your-domain.com/api/user-search?t=@company.com
```
